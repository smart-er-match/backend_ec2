		- 퍼블릭ip 43.203.116.205, 프라이빗ip 172.31.16.191

	프론트:
		- ec2(ubuntu 22.04 t3-medium
		- ssh는 개발자 ip만 열어두고 http, https 는 전체오픈(서비스 할것이기 때문)
		- 서비스할 도메인은 구매할 지 안할지 고려중

		- 모바일에서 apk 다운받고 빈껍데기에 url만 접속하게 해서 동적배포, 자동배포할예정(dev로 실시간 자동화 배포)

추가 보완 메모 (기존 내용에 더해 명세/ERD/PRD/아키텍처 준비용)
- 데이터 수집 주기 명시: 실시간 가용병상 5분, 위치/기본 정보 24시간. 실패 시 재시도(백오프)와 로그/알림(슬랙 웹훅 등) 필요.
- 백업/복구: 백엔드 EC2 내 Postgres 스냅샷 일 1회 목표. 장애 시 복구 절차 별도 문서화 예정.
- 로그/모니터링 최소셋: 로그인/인증 시도, 외부 API 호출 실패, 병상 동기화 성공/실패 건수. PII 마스킹 검토.
- 계정 정책: 이메일과 소셜 계정 통합은 하지 않음. 이메일+소셜 중복 방지 규칙 정의 필요.
- 권한: 기본 GENERAL, 구급요원 인증 승인 후 PARAMEDIC. 관리자 승인 플로우/대시보드 요구사항 추후 명세에 포함.
- 검색/추천 로직 초안: 거리 필터(기본 10km 이상), 진료과 매핑(LLM 또는 룰셋), 병상 가용 정의(hv 필드 중 어떤 값으로 “가능” 판단할지) 설정 필요.
- 지도 딥링크: 네이버/티맵/카카오 맵 우선, 인앱 뷰가 안되면 외부 앱 열기. 경로 요청시 TMAP/카카오 길찾기 API 사용 여부 결정.
- 보안: SSH 제한 유지, HTTP는 프론트 보안그룹만 허용. 개인정보(이메일/휴대폰) 보관/삭제 정책, 탈퇴 처리 절차 정의 필요.
- 테스트/배포: dev/stage/prod 분리, APK 웹뷰 동적 URL 교체 시 버전 관리와 롤백 절차 필요. 크론/배치 헬스체크 포함.
- 향후 산출물 체크리스트: 명세서(로그인/소셜/구급요원 인증/추천/지도/관리자 승인), ERD(User/ParamedicInfo/Hospital/Status/Severe/Location/Department/SymptomMapping/SearchLog 등), PRD(KPI: 추천 응답 95p, 데이터 신선도), 아키텍처(프론트↔백엔드↔DB, 크론 5분/24시간, 소셜/지도/LLM 연동, 로깅·알림 경로).
