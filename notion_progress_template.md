## 기록 전 확인 명령
1. `git status -sb` — 변경 파일 유무 확인
2. `git log -1 --oneline` — 기준 커밋과 비교할 최신 기록 확인
3. `git diff --stat HEAD` 또는 필요한 파일 단위 diff — Notion에 남길 핵심 변경 포인트 추출

## Notion 기록 절차
1. `backend_progress` 데이터베이스에서 해당 날짜(DAY_XX) 페이지 생성
2. `progress_summation` 필드에 하루 전체를 1문장 요약 (100자 내외)
3. 페이지 본문에 불릿 8~10개 이내로, **당일 새로 진행한 작업만** 영역별 정리  
   - 인프라/보안, 시스템 패키지, 애플리케이션/가상환경, 배포 구성, 접근/툴링, 검증/로그 등
4. Codex CLI 또는 VS Code SSH 원격 세션을 활용했다면 연결 형태(포트 포워딩, MCP 연동 여부)도 명시
